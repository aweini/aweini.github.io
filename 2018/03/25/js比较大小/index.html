=== 和 ==

=== 严格运算符
== 相等运算符

相等运算符在比较相同类型的数据时，与严格相等运算符完全一样。
在比较不同类型的数据时，相等运算符会先将数据进行类型转换(隐式转换)，然后再用严格相等运算符比较。

严格运算符的运算规则如下
(1)不同类型值
如果两个值的类型不同，直接返回false。
(2)同一类的原始类型值
同一类型的原始类型的值（数值、字符串、布尔值）比较时，值相同就返回true，值不同就返回false。
(3)同一类的复合类型值
两个复合类型（对象、数组、函数）的数据比较时，不是比较它们的值是否相等，而是比较它们是否指向同一个对象。
null === undefined // false

相等运算符 隐式转换规则
— 如果一个值是null，另一个值是undefined，则它们相等。 null == undefined // true
— 如果一个值是数字，一个值是字符串，先将字符串转换为数字，然后使用转换后的值进行比较。
— 如果其中一个值是true，则将其转换为1在进行比较。如果其中一个值是false，则将其转换为0再进行比较
— 如果其中一个值是对象，另一个值是数字或字符串，会将对象转换为原始值（通过 toString（）或 valueOf（）方法获取），然后比较。
valueOf() 方法可返回 Boolean 对象的原始值。
‘abc’ 和 123 比较怎么都是false
